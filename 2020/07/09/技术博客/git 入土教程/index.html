git 入土教程

1. git 结构

工作区：本地磁盘目录

暂存区：虚拟仓库

git add 文件名
git add * 提交所有文件

git commit -m “这一次提交的描述”

如果回复 nothing to commit, working tree clean就成功了。

修改文件后再次用 git status 查询，会提示有修改。

git log 查看所有提交记录

git reset --hard 版本号（log查询的） 恢复到指定版本

远程仓库：GitHub

暂存区提交到远程仓库，需要先设置 SSH 密钥。

ssh-keygen -t rsa -C "git邮箱"

复制密钥到 GitHub

查看远程仓库：git remote -v

提交到远程仓库：git push -u origin master。如果需要强制提交，在后面加上 -f。

从远程仓库克隆到本地：git clone，这个操作不需要 git init，但会让你输入账号密码。

从远程仓库更新到本地：git pull，需要本地新建一个 git 库。

2. git 的坑

代理

查看当前是否有使用代理

$ git config --global http.proxy

如果返回了端口，而代理已经关闭，则取消

$ git config --global --unset http.proxy

如果需要使用代理，则复制计算机端口配置

git config --global http.proxy http://地址:端口

凡事不成，找 控制面板，删钥匙。
